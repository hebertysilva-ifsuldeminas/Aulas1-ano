O que são Botões?

São o principal meio de interação do usuário, indicando uma ação.

Classes Fundamentais

Classe base: .btn (Obrigatória para aplicar estilos Bootstrap).

Classe de estilo: .btn-* (Define a cor e o significado, ex: .btn-primary).

Uso Semântico

Podem ser aplicados em tags <button>, <a> (links) e <input> (campos de formulário).

Boas Práticas com Botões

Consistência

Use as cores de forma consistente. Ação primária sempre com a mesma cor, ações de exclusão sempre em vermelho, etc.

Clareza

Use textos claros e que indiquem a ação. "Salvar Alterações" é melhor que "Enviar".

Acessibilidade

Para botões com apenas ícones, use a classe .visually-hidden para adicionar um texto descritivo para leitores de tela.

Imagens Responsivas

O desafio de fazer imagens se adaptarem a qualquer tela.

O Problema

Imagens com largura fixa (ex: width="800px") "quebram" o layout em telas menores, criando barras de rolagem indesejadas.

A Solução Bootstrap

Use a classe .img-fluid.

Ela aplica max-width: 100%; e height: auto;, garantindo que a imagem nunca ultrapasse o contêiner pai, adaptando-se perfeitamente.

Boas Práticas com Imagens

Performance

Sempre otimize suas imagens antes de enviá-las para o site. Ferramentas como Squoosh ou TinyPNG podem reduzir drasticamente o tamanho do arquivo sem perder qualidade.

Acessibilidade (Atributo `alt`)

O atributo alt é crucial. Descreva a imagem para leitores de tela e para o caso da imagem não carregar. É importante para SEO e para acessibilidade.

O que são Cards?

São contêineres de conteúdo flexíveis e auto-contidos.

Propósito

Perfeitos para agrupar informações relacionadas, como um post de blog, um produto ou um perfil de usuário, em uma única unidade visual.

Anatomia Básica

.card: O contêiner principal.

.card-body: Onde a maior parte do conteúdo reside.

Outros elementos comuns: .card-title e .card-text.

Resumo da Aula

Botões

Para todas as ações, use a classe base .btn e adicione uma cor com .btn-primary.

Imagens

Para responsividade instantânea, use a classe mágica .img-fluid.

Cards

Para agrupar qualquer conteúdo, use a estrutura principal com .card e .card-body.

Código: Botões Sólidos
<!-- Botões Sólidos -->
<button type="button" class="btn btn-primary">Primary</button>
<button type="button" class="btn btn-secondary">Secondary</button>
<button type="button" class="btn btn-success">Success</button>
<button type="button" class="btn btn-danger">Danger</button>
Código: Botões Outline
<!-- Botões Outline -->
<button type="button" class="btn btn-outline-primary">Primary</button>
<button type="button" class="btn btn-outline-secondary">Secondary</button>
<button type="button" class="btn btn-outline-success">Success</button>
<button type="button" class="btn btn-outline-danger">Danger</button>
Código: Tamanhos de Botões
<!-- Botões com tamanhos diferentes -->
<button type="button" class="btn btn-primary btn-lg">Botão Grande</button>
<button type="button" class="btn btn-secondary">Botão Normal</button>
<button type="button" class="btn btn-info btn-sm">Botão Pequeno</button>
Código: Estados de Botões
<!-- Estado ativo para tag 'a' -->
<a href="#" class="btn btn-primary active" role="button">Link Ativo</a>
<!-- Estado desativado para tag 'button' -->
<button class="btn btn-primary" type="submit" disabled>Botão Desativado</button>
<!-- Estado de Carregamento (Loading) -->
<button class="btn btn-primary" type="button">
  <span class="spinner-border spinner-border-sm"></span>
  <span role="status"> Loading...</span>
</button>
